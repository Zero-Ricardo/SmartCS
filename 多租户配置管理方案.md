# 多租户配置管理方案

> 编写日期：2026-04-03

---

## 一、需求概述

将 SmartCS 从单租户改造为多租户 SaaS 产品，核心改造点：

1. **用户注册/登录**：用户名 + 密码（去掉邮箱）
2. **向量集合隔离**：每个用户独立集合 `kb_{username}`
3. **LLM 配置可编辑**：管理员后台可配置 OpenAI 兼容的 API Base URL 和 API Key
4. **客服聊天路由**：根据租户自动路由到对应集合

## 二、配置层级设计

### 2.1 三层配置架构

```
┌─────────────────────────────────────────────────────────────┐
│  第一层：系统默认配置 (.env / config.py)                      │
│  - 全局 LLM API Key/Base（平台提供，所有租户共享）             │
│  - Qdrant 连接信息                                          │
│  - 系统级参数（超时、并发限制等）                              │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  第二层：租户配置 (tenant_configs 表)                         │
│  - 自定义 LLM API Key/Base（客户自带，可选）                  │
│  - 向量集合名称（kb_{username}）                             │
│  - 品牌配置（名称、颜色、logo、欢迎语）                        │
│  - 系统提示词（自定义 AI 人设）                               │
└─────────────────────────────────────────────────────────────┘
                              │
                              ▼
┌─────────────────────────────────────────────────────────────┐
│  第三层：运行时参数 (请求参数)                                │
│  - guest_id（访客标识）                                      │
│  - session_id（会话标识）                                    │
└─────────────────────────────────────────────────────────────┘
```

### 2.2 配置优先级

**LLM 调用时**：
```
租户自定义 API Key > 系统默认 API Key
```

**向量检索时**：
```
租户集合 (kb_{username}) > 系统默认集合 (pxb)
```

## 三、数据库模型变更

### 3.1 `admin_users` 表改造

**新增字段**：

| 字段 | 类型 | 说明 |
|------|------|------|
| `username` | `VARCHAR(50)` | 用户名，唯一，仅英文+数字，用于登录和集合命名 |
| `password_hash` | `VARCHAR(255)` | 密码哈希（bcrypt，不加密存储） |

**移除/降级字段**：

| 字段 | 变更 |
|------|------|
| `email` | 改为可选（`nullable=True`），不再用于登录 |

**新增属性方法**：

```python
@property
def collection_name(self) -> str:
    """返回该用户的向量集合名称"""
    return f"kb_{self.username}"
```

### 3.2 新增 `tenant_configs` 表

```sql
CREATE TABLE tenant_configs (
    id              VARCHAR(36) PRIMARY KEY,
    admin_user_id   VARCHAR(36) UNIQUE NOT NULL REFERENCES admin_users(id),
    
    -- LLM 配置（客户自带，可选）
    custom_llm_api_key    VARCHAR(255) NULL,
    custom_llm_api_base   VARCHAR(500) NULL,
    custom_llm_model      VARCHAR(100) NULL,
    
    -- 品牌配置
    brand_name      VARCHAR(100) DEFAULT 'AI 客服',
    welcome_message TEXT DEFAULT '你好，有什么可以帮你？',
    brand_color     VARCHAR(7) DEFAULT '#0fdbaa',
    logo_url        VARCHAR(500) NULL,
    placeholder     VARCHAR(200) DEFAULT '请输入你的问题…',
    quick_prompts   JSONB DEFAULT '[]',
    
    -- 高级配置
    system_prompt   TEXT NULL,  -- 自定义系统提示词
    
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);
```

### 3.3 新增 `tenant_tokens` 表

用于 Widget 集成时的租户识别：

```sql
CREATE TABLE tenant_tokens (
    id              VARCHAR(36) PRIMARY KEY,
    admin_user_id   VARCHAR(36) UNIQUE NOT NULL REFERENCES admin_users(id),
    token           VARCHAR(64) UNIQUE NOT NULL,  -- 格式: tk_{32位hex}
    is_active       BOOLEAN DEFAULT TRUE,
    created_at      TIMESTAMP DEFAULT NOW(),
    updated_at      TIMESTAMP DEFAULT NOW()
);
```

## 四、注册/登录改造

### 4.1 注册接口

**请求字段**：

```typescript
interface RegisterRequest {
  username: string;         // 仅英文+数字，字母开头，3-30位
  password: string;         // 密码
  password_confirm: string; // 确认密码
  invite_code: string;      // 邀请码（必填）
  company_name?: string;    // 公司名称（选填）
}
```

**用户名校验规则**：

```python
import re
USERNAME_PATTERN = re.compile(r'^[a-zA-Z][a-zA-Z0-9]{2,29}$')
# 规则：字母开头，仅含英文+数字，3-30 个字符
```

**密码处理**：

```python
import bcrypt

def get_password_hash(password: str) -> str:
    """密码哈希（bcrypt）"""
    return bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt()).decode("utf-8")

def verify_password(plain_password: str, hashed_password: str) -> bool:
    """密码校验"""
    return bcrypt.checkpw(plain_password.encode("utf-8"), hashed_password.encode("utf-8"))
```

**注册流程**：

1. 校验 `password == password_confirm`
2. 校验 `username` 格式（正则）
3. 校验 `username` 唯一性（查库）
4. 校验 `invite_code` 有效性
5. 创建用户 + 创建默认配额 + 创建租户配置 + 生成 Widget Token
6. 标记邀请码已使用

### 4.2 登录接口

```typescript
  password: string;  // 密码
}
```

## 五、向量集合隔离

### 5.1 集合命名规则

```
kb_{username}
```

- `username` 是管理员注册时的用户名
- 示例：用户 `ricardo` → 集合 `kb_ricardo`
- 用户 `acme2026` → 集合 `kb_acme2026`

### 5.2 集合自动创建

**时机**：用户首次上传文档并入库时

**逻辑**：

```python
# qdrant_service.py
def ensure_user_collection(self, username: str) -> str:
    """确保用户集合存在，返回集合名称"""
    collection_name = f"kb_{username}"
    if not self.collection_exists(collection_name):
        self.create_collection(collection_name)
        self.create_payload_indexes(collection_name)
    return collection_name
```

### 5.3 检索路由

**管理后台场景**（管理员自己测试聊天）：

```
JWT Token → get_current_admin → admin.collection_name → stream_agent(collection_name=kb_{username})
```

**SaaS 场景**（客户的终端用户聊天）：

```
Widget Token → tenant_tokens 表 → admin_user → collection_name → stream_agent(collection_name=kb_{username})
```

```python
def get_llm_config(admin_user: AdminUser) -> dict:
    """获取租户的 LLM 配置（优先使用租户自定义配置）"""
    config = admin_user.tenant_config

        "api_base": config.custom_llm_api_base or settings.openai_api_base,
        "model": config.custom_llm_model or settings.openai_model,
```

## 七、客服聊天路由

### 7.1 现有接口

| 接口 | 鉴权 | 集合 | 场景 |
|------|------|------|------|
| `/api/internal/chat/stream` | X-Internal-Secret | `pxb` | 内部服务调用（保留） |

### 7.2 新增接口

| 接口 | 鉴权 | 集合 | 场景 |
|------|------|------|------|
| `/api/chat/stream` | X-Widget-Token 或 `?token=` | `kb_{username}` | SaaS 客户终端用户 |
| `/api/admin/chat/stream` | JWT Bearer | `kb_{username}` | 管理员自己测试 |

### 7.3 路由逻辑

```python
@router.post("/chat/stream")
async def tenant_chat_stream(
    request: ChatStreamRequest,
    token: str = Header(alias="X-Widget-Token") | Query(),
):
    # 1. 验证 token
    tenant_token = await db.execute(
        select(TenantToken).where(TenantToken.token == token, TenantToken.is_active == True)
    )
    if not tenant_token:
        raise HTTPException(401, "Invalid token")
    
    # 2. 获取租户信息
    admin_user = await db.get(AdminUser, tenant_token.admin_user_id)
    collection_name = admin_user.collection_name
    
    # 3. 获取 LLM 配置
    llm_config = get_llm_config(admin_user)
    
    # 4. 调用 RAG 流程
    async for event in stream_agent(
        query=request.query,
        session_id=request.session_id,
        history=request.history,
        collection_name=collection_name,
        llm_config=llm_config,
    ):
        yield event
```

## 十、注意事项

1. **密码安全**：使用 bcrypt 哈希，不存储明文密码
2. **用户名规范**：仅英文+数字，字母开头，3-30位，确保集合名合法
3. **配置优先级**：租户自定义配置 > 系统默认配置
4. **向后兼容**：保留 `/api/internal/*` 接口，不影响现有 Java 网关调用
5. **集合懒创建**：用户注册时不创建集合，首次入库时自动创建
