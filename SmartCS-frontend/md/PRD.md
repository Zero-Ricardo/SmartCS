# 淘课网智能客服前端需求文档 (PRD) - V1.3

## 1. 产品概述

### 产品定位
淘课网的"24小时智能培训顾问"，通过自然语言交互，辅助HR和企业管理者完成从"需求明确"到"选课/找讲师"再到"获取定制方案"的全流程。

### 交互承载形态（强约束）
- 智能客服以前端**弹窗**形态呈现：用户点击主界面的“智能客服”按钮后，弹出悬浮对话窗口。
- 弹窗采用“可展开/可收起”机制，关闭后返回主界面当前上下文，不打断用户原页面浏览。
- 弹窗内完整承载本 PRD 当前定义的功能需求（对话、输入工具、转人工、反馈、历史恢复等）。

### 核心目标
- 缩短用户寻找合适培训产品的时间路径
- 提高高意向客户（如需要定制内训）的线索留存率
- 降低基础人工客服的答疑成本

## 2. 目标用户与核心场景

### 主要用户
- 企业HR
- 培训主管
- 业务部门负责人
- 未注册的潜在采购客户

### 核心场景
- **精准找课**：按行业、岗位、预算搜索公开课或内训课
- **讲师背调**：查询特定领域讲师的资历与评价
- **方案定制**：提交团队培训需求，获取初步的培训规划与报价
- **平台答疑**：咨询开票、合同、退改签等售后规则

## 4. 交互与UI/UX规范

### 视觉风格
- B2B专业风格
- 主色调需匹配淘课网主站橙白风格（品牌橙 + 白底）
- 字体大小适中，留白充足
- 避免过于娱乐化的气泡设计

### 加载状态反馈
- 用户发送消息后立即显示气泡
- AI回复前，展示"智能顾问正在思考中..."的动画

### 消息与链接交互
- 消息内的链接与行动按钮必须有明确的点击反馈
- 外链默认支持新标签页跳转（避免打断对话）
- 若启用联系方式/评价卡片：卡片需具备 Hover/点击反馈与明确的成功/失败提示

### 多语言（i18n）
- **支持语言**：至少支持 `zh-CN`（默认）与 `en-US`，后续可扩展。
- **语言来源与优先级**：用户手动选择 > 本地记忆（LocalStorage/Cookie）> 主站语言设置（如有）> 浏览器语言。
- **覆盖范围**：顶部状态栏、按钮/菜单、空状态/错误提示、表单校验文案、导出聊天记录的固定字段与标题。
- **组件库文案**：Element Plus 相关组件（如 Form 校验、分页、上传等）的 locale 需跟随语言切换。
- **格式化规则**：时间、数字、金额、百分比在不同语言下展示需符合本地化规则（含货币符号与千分位）。
- **AI/人工回复语言**：当用户切换语言后，前端需将 `locale` 隐式携带到请求中，指导后端按对应语言生成回复与卡片文案（若后端支持）。

## 5. 数据与上下文管理

### 页面上下文（Page Context）
前端在每次发送用户消息时，需隐式携带用户当前浏览的商品ID或页面分类。

### 访客与会话管理（完全免登录，支持对话记忆）

#### 游客模式（无需登录）
- 前端在本地（LocalStorage）生成唯一 `Visitor ID`，无需任何登录流程
- 所有对话功能完全开放，用户可直接提问和使用所有功能
- 每次请求在 Header 携带此 ID，拼接多轮对话上下文

#### 对话记忆机制
- **本地存储**：对话历史保存在浏览器本地存储中，重新打开页面时自动恢复
- **持久化会话**：使用 `Visitor ID` 作为会话标识，后端根据此ID维护对话上下文
- **自动续接**：用户重新打开客服窗口时，自动加载最近的对话记录，实现无缝续聊

#### 身份转化（可选）
- 若用户在对话过程中选择登录，前端将 `Visitor ID` 与用户 `Token` 关联
- 登录后可将匿名会话历史合并到用户账户中

## 6. 技术栈定向

### 前端框架
Vue 3 (Composition API + `<script setup>`)

### UI 组件库
Element Plus (按需引入：
- 利用 `el-row/el-col` 实现左右双栏布局
- `el-card` 作为对话框与信息面板的容器（或使用原生布局）
- `el-popover` 实现鼠标悬停展示二维码、emoji 面板等)

### 状态管理
Pinia (管理用户态、访客ID、WebSocket 实例、历史记录及当前服务模式[AI/人工])

### 网络请求
- Axios (常规接口调用)
- Fetch API/原生 WebSocket (处理大模型流式输出)
