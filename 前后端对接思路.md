**关键发现点：**
- 前端用什么方式调用 API（fetch / axios）

### 2. 修改前端 API 地址（关键！）

**问题：** 生产环境不能写死 `http://localhost:8000`，否则用户手机会请求自己手机的 8000 端口

**解决方案：** 使用**相对路径**，让 Nginx 反向代理

```typescript
// 修改前（开发环境）
const API_BASE_URL = "http://localhost:8000";

// 修改后（生产环境）
const API_BASE_URL = ""; // 空字符串 = 相对路径
```

**效果：**
- 请求 `/internal/chat/stream` → 实际请求 `http://用户访问的域名/internal/chat/stream`
- Nginx 拦截 `/internal/*` → 转发到后端容器

### 3. 创建 Nginx 反向代理配置

```nginx
# nginx.conf
server {
    listen 80;
    server_name _;
    root /usr/share/nginx/html;
    index index.html;

    # 前端静态资源（Vue SPA 路由支持）
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理（关键配置）
    location /internal/ {
        # Docker 服务名作为后端地址
        proxy_pass http://smartcs-backend:8000;

        # HTTP 版本和头信息
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;

        # ===== SSE 流式响应必须配置 =====
        # 关闭代理缓存，否则 SSE 会一次性全吐出来
        proxy_buffering off;
        proxy_cache off;
        proxy_request_buffering off;

        # SSE 长连接超时设置
        proxy_read_timeout 300s;
        proxy_send_timeout 300s;
    }

    # 健康检查
    location /health {
        proxy_pass http://smartcs-backend:8000;
        proxy_buffering off;
    }
}
```

**为什么需要 `proxy_buffering off`？**
- SSE 是流式响应，需要实时传输每个 token
- 如果开启缓存，Nginx 会等响应完成才一次性发给浏览器
- 用户就看不到"打字机效果"了

### 4. 优化 Dockerfile（多阶段构建）

```dockerfile
# ===== 第一阶段：构建 =====
FROM node:20-alpine AS builder

WORKDIR /app

# 安装 pnpm
RUN corepack enable && corepack prepare pnpm@9.15.4 --activate

# 安装依赖（利用缓存）
COPY package.json pnpm-lock.yaml* ./
RUN pnpm install --frozen-lockfile

# 复制源代码并构建
COPY . .
ARG VITE_API_BASE_URL=
ARG VITE_INTERNAL_SECRET=your-internal-secret-key
ENV VITE_API_BASE_URL=${VITE_API_BASE_URL}
ENV VITE_INTERNAL_SECRET=${VITE_INTERNAL_SECRET}
RUN pnpm build  # 编译 Vue → dist/

# ===== 第二阶段：运行 =====
FROM nginx:1.27-alpine

# 删除默认配置
RUN rm -rf /etc/nginx/conf.d/default.conf /usr/share/nginx/html/*

# 只拷贝编译产物（不含源代码）
COPY --from=builder /app/dist /usr/share/nginx/html

# 复制 nginx 配置
COPY nginx.conf /etc/nginx/conf.d/default.conf

EXPOSE 80
CMD ["nginx", "-g", "daemon off;"]
```

**多阶段构建的优势：**
| 阶段 | 镜像 | 大小 | 内容 |
|------|------|------|------|
| 第一阶段 | node:20-alpine | ~500MB | Node.js + 源代码 + 依赖 |
| 第二阶段 | nginx:alpine | ~25MB | 只有 Nginx + 静态文件 |

最终镜像只有 Nginx + 静态文件，**不含源代码**，更安全更小。

### 6. 编写 docker-compose.yml

```yaml
services:
  # ===== 前端：Nginx 静态服务 =====
  smartcs-frontend:
    build:
      context: ./SmartCS-frontend
      dockerfile: Dockerfile
    container_name: smartcs-frontend
    restart: unless-stopped
    ports:
      - "3000:80"  # 对外暴露 3000 端口
    depends_on:
      smartcs-backend:
        condition: service_healthy  # 等待后端健康检查通过
    networks:
      - smartcs-network

  # ===== 后端：FastAPI 服务 =====
  smartcs-backend:
    build:
      context: .
      dockerfile: Dockerfile
    container_name: smartcs-backend
    restart: unless-stopped
    environment:
      - DATABASE_URL=postgresql+psycopg://postgres:postgres@postgres:5432/smartcs
      - QDRANT_HOST=qdrant
      - QDRANT_PORT=6333
    depends_on:
      postgres:
        condition: service_healthy
      qdrant:
        condition: service_started
    healthcheck:
      test: ["CMD", "python", "-c", "import urllib.request; urllib.request.urlopen('http://localhost:8000/health')"]
      interval: 10s
      timeout: 5s
      retries: 5
      start_period: 30s
    networks:
      - smartcs-network

  # ===== PostgreSQL 数据库 =====
  postgres:
    image: postgres:16
    container_name: smartcs-postgres
    restart: unless-stopped
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: postgres
      POSTGRES_DB: smartcs
    ports:
      - "5432:5432"
    volumes:
      - postgres_data:/var/lib/postgresql/data
    healthcheck:
      test: ["CMD-SHELL", "pg_isready -U postgres"]
      interval: 5s
      timeout: 5s
      retries: 5
    networks:
      - smartcs-network

  # ===== Qdrant 向量数据库 =====
  qdrant:
    image: qdrant/qdrant:latest
    container_name: smartcs-qdrant
    restart: unless-stopped
    ports:
      - "6333:6333"
      - "6334:6334"
    volumes:
      - qdrant_data:/qdrant/storage
    networks:
      - smartcs-network

# 容器间通信网络
networks:
  smartcs-network:
    driver: bridge

# 数据持久化
volumes:
  postgres_data:
  qdrant_data:
```

**关键配置说明：**
- `networks`：所有容器在同一网络，可以用**服务名**互相访问（如 `postgres:5432`）
- `depends_on + healthcheck`：确保启动顺序正确
- `volumes`：数据持久化，容器重启数据不丢失

## 三、前端如何调用后端 API

### 1. SSE 流式请求

```typescript
// chatApi.ts
export const sendMessageStream = async (params, handlers) => {
  const response = await fetch(`/internal/chat/stream`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "X-Internal-Secret": "your-internal-secret-key"  // S2S 鉴权
    },
    body: JSON.stringify({
      query: params.content,
      guest_id: params.visitorId,
      session_id: params.sessionId
    })
  });

  // 获取流式 reader
  const reader = response.body.getReader();
  const decoder = new TextDecoder();

  let buffer = "";
  while (true) {
    const { value, done } = await reader.read();
    if (done) break;

    buffer += decoder.decode(value, { stream: true });

    // SSE 用 \n\n 分割事件块
    const blocks = buffer.split("\n\n");
    buffer = blocks.pop() ?? "";  // 保留不完整的块

    for (const block of blocks) {
      const parsed = parseEventBlock(block.trim());
      if (!parsed) continue;

      // 根据事件类型分发
      if (parsed.event === "session") {
        handlers.onSession(JSON.parse(parsed.data).session_id);
      } else if (parsed.event === "citations") {
        handlers.onCitations(JSON.parse(parsed.data));
      } else if (parsed.data === "[DONE]") {
        handlers.onDone();
      } else {
        handlers.onToken(parsed.data);
      }
    }
  }
};
```

### 2. SSE 事件解析

**后端返回的 SSE 格式：**
```
event: session
data: {"session_id": "xxx-xxx-xxx"}

data: 培训

data: 宝

data: 是

event: citations
data: [{"source": "https://...", "doc_title": "xxx.md", "score": 0.8}]

data: [DONE]
```

**前端解析逻辑：**
```typescript
const parseEventBlock = (block: string) => {
  const lines = block.split("\n");
  let event = "message";  // 默认事件类型
  const dataLines: string[] = [];

  for (const line of lines) {
    if (line.startsWith("event:")) {
      event = line.slice(6).trim();
    }
    if (line.startsWith("data:")) {
      let payload = line.slice(5);
      if (payload.startsWith(" ")) {
        payload = payload.slice(1);
      }
      dataLines.push(payload);
    }
  }

  if (dataLines.length === 0) return null;

  return {
    event,
    data: dataLines.join("\n")
  };
};
```

## 四、对接检查清单

### 后端检查
- [ ] API 路径是否有统一前缀（如 `/internal/` 或 `/api/`）
- [ ] 是否有 CORS 配置（开发环境需要）
- [ ] SSE 接口是否正确返回 `event:` 和 `data:` 格式
- [ ] 健康检查端点 `/health` 是否可用
- [ ] 鉴权方式是否明确（Header / Cookie / Token）

### 前端检查
- [ ] API 地址是否使用相对路径（生产环境）
- [ ] SSE 事件解析是否正确处理 `event:` 类型
- [ ] 是否有 pending 状态显示加载动画
- [ ] 错误处理是否完善（网络错误、超时、业务错误）
- [ ] session 管理是否正确（新建、复用、过期）

### Nginx 检查
- [ ] `proxy_pass` 地址是否正确（Docker 服务名或 IP）
- [ ] `proxy_buffering off` 是否配置（SSE 必须）
- [ ] 超时时间是否足够（SSE 长连接）
- [ ] WebSocket 支持是否配置（如有需要）

### Docker 检查
- [ ] 所有服务是否在同一 `networks`
- [ ] `depends_on` 启动顺序是否正确
- [ ] `healthcheck` 是否配置
- [ ] `volumes` 数据持久化是否配置
- [ ] 端口映射是否正确（避免冲突）

## 五、常见问题与解决方案

| 问题 | 原因 | 解决方案 |
|------|------|---------|
| SSE 一次性输出 | Nginx 缓冲 | `proxy_buffering off` |
| 跨域错误 | CORS 未配置 | 后端添加 `CORSMiddleware` |
| 容器间无法通信 | 不在同一网络 | 添加到同一个 `networks` |
| 启动顺序错误 | 依赖未就绪 | 添加 `healthcheck` + `depends_on` |
| 前端请求 404 | 路径不匹配 | 检查 Nginx `location` 配置 |
| 前端路由刷新 404 | SPA 路由未处理 | `try_files $uri $uri/ /index.html` |
| 数据丢失 | 未配置持久化 | 添加 `volumes` |
| API 地址请求到本地 | 写死了 localhost | 使用相对路径或环境变量 |

## 六、开发 vs 生产环境配置对比

| 配置项 | 开发环境 | 生产环境 |
|--------|---------|---------|
| 前端 API 地址 | `http://localhost:8000` | `""`（相对路径） |
| 后端监听地址 | `127.0.0.1:8000` | `0.0.0.0:8000` |
| CORS | 允许所有来源 | 限制来源 |
| 热加载 | 开启 | 关闭 |
| 日志级别 | DEBUG | INFO/ERROR |
| 数据库 | 本地或 Docker | 云数据库或独立容器 |

## 七、快速对接模板

### 最小化 Nginx 配置
```nginx
server {
    listen 80;
    root /usr/share/nginx/html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    location /api/ {
        proxy_pass http://backend:8000;
        proxy_buffering off;
    }
}
```

### 最小化 docker-compose.yml
```yaml
services:
  frontend:
    build: ./frontend
    ports: ["80:80"]
    depends_on: [backend]

  backend:
    build: ./backend
    ports: ["8000:8000"]

networks:
  default:
    driver: bridge
```
