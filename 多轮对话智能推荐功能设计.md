# 智能推荐功能：课程/讲师推荐

> 编写日期：2026-04-03

---

## 一、背景与目标

**背景：** 当前 SmartCS 只做 RAG 问答（从帮助文档向量库检索答案）。用户希望扩展：当用户询问课程或讲师时，AI 能主动从 Java 网关查询结构化数据并推荐。

**决策：**
- 架构：扩展现有 `/internal/chat/stream` 聊天流，不新建独立接口
- 数据来源：Java 网关 REST API（后续对接，暂时用 mock 数据）

**核心流程：** 用户问题 → `analyze_node` 判断 intent → 如果是推荐类问题 → `recommend_node` 调用 Java 网关查询 → SSE `recommendations` 事件返回前端

---

## 二、方案设计

### 2.1 intent 扩展

`analyze_node` 中新增 `recommend` intent：

```python
# 闲聊判断（保持不变）
chat_keywords = ["你好", "在吗", "你是谁", ...]
if len(query) < 15 and any(kw in query.lower() for kw in chat_keywords):
    return {"intent": "direct_answer", ...}

# 推荐类问题判断
recommend_keywords = ["课程", "讲师", "培训", "老师", "报名", "推荐", ...]
if any(kw in query for kw in recommend_keywords):
    return {"intent": "recommend", "recommend_type": "course"}  # 或 "instructor"

# 其他走 RAG
return {"intent": "need_rag", ...}
```

### 2.3 graph.py 修改

`stream_agent` 中新增推荐分支：

```python
if intent == "recommend":
    rec_result = await recommend_node({"query": query, "recommend_type": recommend_type})
    yield {
        "type": "recommendations",
        "data": rec_result["recommendations"],
        "recommend_type": recommend_type
    }
    yield {"type": "done"}
    return  # 不走 RAG 和 LLM 生成
```
