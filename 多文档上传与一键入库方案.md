# 多文档上传与一键入库方案

> 编写日期：2026-04-03

---

## 一、现状分析

### 1.1 当前功能

- **上传**：单文件上传，使用 `el-upload` 组件，每次只能选一个文件
- **入库**：单文档入库，每行有一个"处理"按钮，点击后启动解析+入库管线
- **进度轮询**：每 2 秒轮询正在处理的文档状态

### 1.2 问题

1. 批量上传 10 个文件需要点 10 次上传按钮
2. 批量入库 10 个文档需要点 10 次"处理"按钮
3. 效率低下，用户体验差

---

## 二、方案设计

### 2.1 多文档上传

**前端改造**：
- `el-upload` 组件添加 `multiple` 属性，支持一次选择多个文件
- 上传时并发上传（或串行上传，避免服务器压力）

**后端改造**：
- 新增批量上传接口 `POST /admin/knowledge/upload/batch`
- 接收多个文件，返回上传结果列表

### 2.2 一键入库

**前端改造**：
- 表头添加复选框列，支持多选
- 新增"批量处理"按钮，对选中的文档执行入库
- 显示批量处理进度（X/总）

**后端改造**：
- 新增批量处理接口 `POST /admin/knowledge/documents/batch-process`
- 接收文档 ID 列表，依次启动处理管线

const selectedDocs = ref<KnowledgeDocument[]>([]);

const handleSelectionChange = (selection: KnowledgeDocument[]) => {
  selectedDocs.value = selection;
};

const handleBatchProcess = async () => {
  // 只处理 uploaded/parsed/failed 状态的文档
  const toProcess = selectedDocs.value.filter(
    doc => ['uploaded', 'parsed', 'failed'].includes(doc.status)
  );
  
  if (toProcess.length === 0) {
    ElMessage.warning("没有可处理的文档");
    return;
  }
  
  try {
    await batchProcessDocuments(toProcess.map(d => d.id));
    ElMessage.success(`已启动 ${toProcess.length} 个文档的处理任务`);
    
    // 将这些文档加入轮询
    for (const doc of toProcess) {
      doc.status = "parsing";
      pollingIds.value.add(doc.id);
    }
    updatePolling();
  } catch (e: any) {
    ElMessage.error(e.message || "批量处理失败");
  }
};

### 3.2 后端改造

#### 3.2.1 批量上传接口

```python
@router.post("/upload/batch", response_model=List[DocumentResponse])
async def upload_documents_batch(
    files: List[UploadFile] = File(...),
    current_admin: AdminUser = Depends(get_current_admin),
    db: AsyncSession = Depends(get_db)
):
    """批量上传知识库文档"""
    results = []
    
    for file in files:
        try:
            # 复用单文件上传逻辑
            doc = await _upload_single_file(file, current_admin, db)
            results.append(doc)
        except Exception as e:
            # 记录失败但继续处理其他文件
            results.append({"filename": file.filename, "error": str(e)})
    
    return results
```

#### 3.2.2 批量处理接口

```python
@router.post("/documents/batch-process")
async def batch_process_documents(
    request: BatchProcessRequest,  # { doc_ids: List[str] }
    background_tasks: BackgroundTasks,
    current_admin: AdminUser = Depends(get_current_admin),
    db: AsyncSession = Depends(get_db)
):
    """批量启动文档处理管线"""
    
    # 验证文档归属
    query = select(KnowledgeDocument).where(
        KnowledgeDocument.id.in_(request.doc_ids),
        KnowledgeDocument.admin_user_id == current_admin.id
    )
    result = await db.execute(query)
    docs = result.scalars().all()
    
    if not docs:
        raise HTTPException(status_code=404, detail="No documents found")
    
    # 过滤出可处理的文档
    processable = [d for d in docs if d.status in ["uploaded", "parsed", "failed"]]
    
    # 依次启动后台任务
    for doc in processable:
        doc.status = "parsing" if doc.file_type in ["pdf", "docx", "txt"] else "ingesting"
        doc.ingest_progress = 0
        doc.error_message = None
        background_tasks.add_task(process_document_pipeline, doc.id)
    
    await db.commit()
    
    return {
        "message": f"已启动 {len(processable)} 个处理任务",
        "processed_count": len(processable),
        "skipped_count": len(docs) - len(processable)
    }
```

## 五、简化方案（推荐）

考虑到批量上传接口需要改动较多，可以先实现**一键入库**功能，上传保持单文件但支持连续选择。

### 5.1 简化版上传

`el-upload` 的 `multiple` 属性本身支持多选，但会为每个文件触发一次 `http-request`。我们可以：

1. 保持单文件上传接口不变
2. 前端支持多选，每个文件独立上传
3. 显示上传进度（如"正在上传 3/10"）

### 5.2 一键入库

这是核心需求，优先实现：
- 表格添加复选框
- 新增"批量处理"按钮
- 后端新增批量处理接口
